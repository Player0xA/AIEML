# Drag-and-Drop Bug Fix - Test Report

## Issue
The web UI claimed to accept .eml files but the JavaScript only processed .json files. Users would drop .eml files and get no response or cryptic errors.

## Fix Applied
Modified `handleFiles()` function in `app.js` to:
1. **Detect file types** before processing
2. **EML files**: Show helpful toast + alert directing users to CLI
3. **JSON files**: Process normally as emltriage output
4. **Mixed files**: Block with clear error message

## Implementation

### Code Changes (app.js:380-461)
```javascript
function handleFiles(files) {
  const fileList = Array.from(files);
  
  // Separate JSON and EML files
  const jsonFiles = fileList.filter(f => f.name.endsWith('.json'));
  const emlFiles = fileList.filter(f => f.name.endsWith('.eml'));
  
  // Handle EML files - show helpful message
  if (emlFiles.length > 0) {
    const names = emlFiles.map(f => f.name).join(', ');
    showToast(`EML files detected (${names}). Please analyze with CLI first: emltriage analyze file.eml -o ./output`, 'error', 8000);
    
    setTimeout(() => {
      alert(`📧 EML File Detected: ${names}\n\nTo analyze .eml files:\n1. Use CLI: emltriage analyze ${emlFiles[0].name} -o ./output\n2. Then load the generated artifacts.json here\n\nOr use the 'Load Demo Data' button to see how it works.`);
    }, 100);
    
    return;
  }
  
  if (jsonFiles.length === 0) {
    showToast('Please select JSON files (artifacts.json, iocs.json, etc.)', 'error');
    return;
  }
  
  // Process JSON files normally...
}
```
